## Step 2 — 디자인 업그레이드

```
방금 만든 민원 데이터 뷰어를 step2/index.html로 복사해서 디자인을 업그레이드해줘.

변경사항:
1. 공공기관 파란색 테마 전면 적용 (primary: #1a3a6b)
2. CSS 변수(:root) 시스템으로 색상 관리
3. 상단에 통계 카드 4개 추가: 전체건수, 긴급, 보통, 낮음 (아직 데이터는 비활성)
4. 카드, 테이블, 업로드 영역에 둥근 모서리(12px)와 그림자 효과
5. 학습 포인트 박스에 "CSS 변수란?" 설명 추가
6. PRD 비교 카드: Step 1 PRD vs Step 2 PRD를 나란히 보여주기

디자인만 바꾸고 기능은 Step 1과 동일하게 유지해줘.
폰트: 맑은 고딕, 배경: #eef2f7
```

## Step 3 — AI 분류 기능 추가

```
step2/index.html을 step3/index.html로 복사해서 Claude AI 민원 분류 기능을 추가해줘.

추가할 기능:
1. API 키 입력창 + 비밀번호 토글 + 모드 뱃지 (AI모드/키워드모드)
2. API 키 localStorage 저장 (키: 'ftc_api_key')
3. "🤖 AI 분류 시작" 버튼
4. 진행률 바: "분류 중... 45/200"

AI 분류 로직:
- API 키 있을 때: Claude API로 각 민원을 1건씩 분류
  - 엔드포인트: https://api.anthropic.com/v1/messages
  - 모델: claude-sonnet-4-5-20250929
  - 헤더에 'anthropic-dangerous-direct-browser-access': 'true' 필수
  - 시스템 프롬프트로 JSON 형식 응답 요청:
    {"유형","담당부서","우선순위","요약","관련법령"}
- API 키 없을 때: 키워드 매칭 자동 분류
  - 담합→카르텔조사국, 불공정→시장감시국, 광고→소비자정책국 등

테이블 컬럼 추가: 유형 | 담당부서 | 우선순위 | 요약 | 관련법령
우선순위 뱃지: 긴급(빨강), 보통(노랑), 낮음(초록)
통계 카드 4개를 분류 결과로 실시간 업데이트

학습 포인트:
- "API Key란?" 개념 카드
- "시스템 프롬프트란?" 개념 카드
- AI 로그 콘솔 (어두운 배경에 색깔별 로그 표시)

API 키 기본값: 여기에-교육용-API-키-입력
```

- 출장지 (필수)
- 출장 목적 (필수, textarea)
- 주요 활동 내용 (필수, textarea, 줄바꿈으로 구분)

- 결재란 테이블 (기안/검토/결재)

- A4 세로

API 키 기본값: 여기에-교육용-API-키-입력

## 교육 팁

### 강사 진행 순서
1. Step 1: "PRD를 넣으면 이런 결과가 나옵니다" (PRD의 중요성)
2. Step 2: "채팅으로 디자인만 바꿔달라고 했더니..." (점진적 개선)
3. Step 3: "AI를 연결하니 이렇게 됩니다" (AI API 연동의 쉬움)
4. Step 4: "차트도 추가해달라고 하면..." (시각화)
5. Step 5: "최종 완성본, 실제 업무에 쓸 수 있습니다" (임팩트)
6. 템플릿: "실제 공정위 업무 문서도 만들 수 있어요" (확장성)

### 안티그래비티 사용 시 주의
- 각 Step은 이전 Step 코드가 있는 상태에서 진행
- 프롬프트를 한 번에 다 넣기 (중간에 끊지 않기)
- 오류 발생 시 "오류 수정해줘" + 오류 메시지 붙여넣기
- API 키는 교육용이므로 교육 후 삭제 예정
