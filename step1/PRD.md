# 공정거래위원회 민원 데이터 뷰어 — PRD

## 프로젝트 개요
공정거래위원회에서 접수된 민원 엑셀 파일을 업로드하면 웹 테이블로 보여주는 단일 HTML 페이지 앱입니다.

## 기술 스택
- 순수 HTML/CSS/JavaScript (단일 index.html 파일)
- 외부 라이브러리: SheetJS (CDN) — 엑셀 파일 파싱용
```html
<script src="https://cdnjs.cloudflare.com/ajax/libs/xlsx/0.18.5/xlsx.full.min.js"></script>
```
- 빌드 도구 없음, 브라우저에서 바로 실행

## 디자인
- 헤더: 배경 #1a3a6b (공공기관 파란색), 흰색 글자
- 헤더에 🏛️ 아이콘 + "민원 데이터 뷰어" 제목 + "STEP 1 — 기본 업로드" 뱃지
- 본문 배경: #f5f5f5
- 폰트: '맑은 고딕' (Malgun Gothic)
- 전체 최대 폭: 1100px, 가운데 정렬

## 기능 요구사항

### 1. 파일 업로드 영역
- 점선 테두리(dashed) 드래그&드롭 영역
- 클릭해도 파일 선택 가능
- 지원 포맷: .xlsx, .xls, .csv
- 파일 선택 후 파일명 표시
- 📁 아이콘 + "엑셀 파일을 클릭하거나 드래그하여 업로드하세요" 안내 문구

### 2. 건수 표시 바
- 파일 업로드 후 나타남 (초기에는 숨김)
- "📋 총 민원: N건" 형태로 큰 숫자 강조

### 3. 데이터 테이블
- 컬럼: 번호 | 제목 | 민원내용 | 접수일
- 헤더 행 배경: #e8ecf1
- 마우스 오버 시 행 하이라이트
- 민원내용 컬럼: 긴 텍스트는 말줄임(...) 처리, 마우스 오버 시 전체 내용 tooltip
- 업로드 전: "엑셀 파일을 업로드하면 여기에 데이터가 표시됩니다." 안내 메시지

### 4. 학습 포인트 박스 (교육용)
노란 배경(#fffbea) + 왼쪽 주황 보더로 된 학습 안내 박스를 4개 배치:

**박스 1 — Step 1의 목표** (업로드 영역 위)
> 이 단계는 "바이브코딩의 첫 결과물"입니다. PRD에 "엑셀을 업로드하면 테이블로 보여줘"라고만 적으면, AI가 이 정도의 동작하는 앱을 만들어줍니다.

**박스 2 — 엑셀 파일은 어떻게 읽을까?** (업로드 영역 아래)
> SheetJS (xlsx.js) 라이브러리를 CDN으로 불러와서 엑셀을 JavaScript 배열로 변환합니다.
> "▶ 코드 보기" 토글 버튼으로 코드 예시를 접었다 펼 수 있게
> 코드 블록(어두운 배경)에 FileReader → XLSX.read → sheet_to_json 흐름 표시

**박스 3 — 데이터를 화면에 표시하기** (건수 바 아래, 파일 업로드 후에만 표시)
> forEach로 JSON 배열을 HTML 테이블 행으로 변환하는 과정 설명
> 토글 코드 블록 포함

**박스 4 — 다음 단계 예고** (테이블 아래)
> Step 2에서 추가할 것: 디자인 변경, AI 분류, 통계 카드
> "점진적으로 요구사항을 추가하는 것이 바이브코딩의 핵심입니다" 메시지

## 엑셀 데이터 형식
엑셀 파일의 첫 번째 시트를 읽으며, 컬럼명은 다음과 같습니다:
- 번호
- 제목
- 민원내용
- 접수일

접수일이 엑셀 날짜 숫자(serial number)인 경우 YYYY-MM-DD 문자열로 변환 처리합니다.

## 주의사항
- AI 기능 없음 (이 단계는 순수 데이터 표시만)
- API 키 입력창 없음
- 차트 없음
- 최대한 심플하게 — 이게 Step 1의 의도
- HTML 특수문자 이스케이프 처리 (XSS 방지)
